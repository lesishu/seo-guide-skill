# 页面优化（On-Page SEO）

## 核心要素清单

### 1. 标题标签（Title Tag）

**最佳实践：**
- 长度：50-60 字符（Google 通常截断超过此长度的标题）
- 结构：`主关键词 - 品牌名` 或 `主关键词 | 辅助关键词 - 品牌名`
- 独特性：每个页面标题唯一，避免重复
- 吸引点击：包含数字、疑问词、情感词

**示例：**
```
✅ 推荐：项目管理软件哪个好？2025年TOP10对比评测 | WorkTool
❌ 避免：项目管理软件 - WorkTool
```

### 2. 元描述（Meta Description）

**最佳实践：**
- 长度：150-160 字符
- 包含目标关键词（加粗/自然出现）
- 清晰描述页面内容 + 明确 CTA
- 独特性：每个页面唯一

**示例：**
```
✅ 推荐：想找一款好用的项目管理软件？本文对比了2025年最受欢迎的10款工具，
   涵盖功能、价格、适用场景，帮你快速选到合适的。立即查看完整评测。
❌ 避免：项目管理软件是现代企业必备的工具，欢迎了解我们的产品。
```

### 3. URL 结构

**最佳实践：**
- 简短、可读：`/project-management-software-review/` 而非 `/p=123`
- 包含目标关键词
- 使用连字符 `-` 分隔词（不用下划线 `_`）
- 避免无意义参数

### 4. 标题层级（H1-H6）

- **H1：** 页面唯一，核心主题，通常与 Title 一致
- **H2：** 主要章节标题，包含次要关键词
- **H3：** 子章节
- 避免跳过层级（H1 → H3）
- 避免一个 H2 下只有一句话

### 5. 图片优化

| 要素 | 说明 |
|------|------|
| 文件名 | 描述性：`project-management-tool.jpg` 而非 `IMG_1234.jpg` |
| Alt 文本 | 描述图片内容 + 包含关键词（但不要堆砌）|
| 尺寸/格式 | WebP/AVIF > JPG/PNG；压缩至 100KB 以下 |
| 宽度 | 响应式，最大宽度不超过容器宽度 |
| 懒加载 | 添加 `loading="lazy"` |

### 6. 内链（Internal Links）

**策略：**
- 上下文内链 > 文末相关文章
- 锚文本描述目标页面内容（不用"点击这里"）
- 每个页面至少 2-3 个内链
- 重要页面应有更多内链支持（链接 equity 传递）

**示例：**
```
✅ 推荐：了解我们的「任务管理功能」如何提升团队效率。
❌ 避免：想了解更多，请点击这里。
```

### 7. 内容质量

- **E-E-A-T 注意：** E-E-A-T 是 Google 质量评估员（Search Quality Rater）用来评估内容质量的**框架**，**不是排名因素**。与其刻意满足 E-E-A-T，不如专注写有帮助、可信赖、专业的内容。Google 官方明确说明："Expect your readers' search terms. Think about the words that a user might search for to find a piece of your content."

- **内容深度：** 覆盖用户需求的完整程度（Search Quality Rater Guidelines 的 Needs Met 评级）
- **内容长度：** 无硬性要求，能完整回答用户意图即可（通常 1000-3000 字）
- **更新频率：** 定期更新过时内容

### 8. 内容可读性

- 段落简短（3-4 句以内）
- 使用列表、表格、图表
- 适当使用图片/截图辅助说明
- Flesch Reading Ease：60-70（中文页面对应白话文即可）
